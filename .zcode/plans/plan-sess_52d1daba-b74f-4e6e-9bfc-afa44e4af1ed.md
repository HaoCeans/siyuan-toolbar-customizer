## 图标选择器增加搜索功能方案

### 改动范围
仅改 **1 个文件**：`src/ui/iconPicker.ts`（约 +80 行 / -20 行）

### 数据结构概览

| 分区 | 可搜索字段 | 示例 |
|------|-----------|------|
| **思源图标** | `description_zh_cn` + `keywords` + `category.title_zh_cn` | 搜"笑脸"匹配😀 |
| **阿里图标** | 文件名(pinyin) + 分类名 | 搜"bangbang"匹配棒棒糖 |
| **本体图标** | emoji 字符 + 分类名 | 搜"表情"匹配分类 |

### 交互流程

```
搜索框空 → 保持现有分类浏览行为（完全不改动）
搜索框有内容 → 
  1. 隐藏一级分区标签（避免干扰），仅搜当前分区
  2. 跨当前分区的所有分类展平搜索
  3. 显示"找到 N 个匹配"的提示
  4. 网格渲染匹配结果
  5. 无匹配时显示"未找到"
清空搜索 → 恢复分区标签和分类标签，回到浏览模式
切换分区时若有搜索词 → 自动清空搜索
弹窗打开 → 自动聚焦搜索框
```

### 具体改动

1. **提取 `createIconButton` 和 `createGrid` 工具函数**（减少重复代码，同时服务于搜索和分类浏览）
2. **新增 `searchKeyword` 变量 + 搜索框 DOM**（input + 🔍图标 + ×清除按钮）
3. **新增 `renderSearchResults()` 函数**（跨分类展平搜索，3 个分区各自实现过滤逻辑）
4. **修改 `updateCategoryTabs`**：搜索模式下不渲染分类标签，显示"找到 N个"状态
5. **分区标签切换时**：若有搜索关键词则自动清空
