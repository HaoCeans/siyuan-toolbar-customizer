## 实现计划：胶囊宽度设置

### 目标
给胶囊模式新增一个「固定宽度」滑块，让用户控制胶囊宽度，溢出计算自动以此为准。

### 改动涉及 4 处

**1. `src/toolbarManager.ts` — 接口 + 默认值 + CSS**

| # | 行 | 改动 |
|---|------|------|
| ① | ~L74 `MobileToolbarConfig` | 加 `floatingToolbarWidth?: string` |
| ② | ~L210 `DEFAULT_MOBILE_CONFIG` | 加 `floatingToolbarWidth: '0'` (0=自动) |
| ③ | ~L1041-1042 CSS 注入 | 根据值决定 `width` / `max-width` |

CSS 改动逻辑：
- 值为 0 / 空 → `width: auto; max-width: 95vw`（当前行为）
- 值 > 0 → `width: ${值}px; max-width: none`

**2. `src/settings/mobile.ts` — UI 滑块**

| # | 行 | 改动 |
|---|------|------|
| ④ | ~L2225 后追加 | 新增 `⑤胶囊自身宽度` 滑块，范围 120~500px，默认 `0`=自动 |

完全复用现有模式：`createCustomSliderWithoutLabel` + `.floating-toolbar-setting` + `updateMobileToolbar()`。

### 为什么不用改其他代码

- `calculateButtonOverflow()` → 它读 `getToolbarAvailableWidth()` (= DOM 的 `getBoundingClientRect().width`)，CSS 变了它自然读到正确值
- `showOverflowToolbar()` → 溢出面板用 `left:10px; right:10px` 全屏定位，与胶囊宽度无关
- 滚动隐藏 → 已用 `toolbarAutoHideCapsuleMode` 标记正确处理

### 默认行为
值设为 `0` 时完全保持现有行为（`width: auto; max-width: 95vw`），不产生任何影响。
